Overview

People lie, cryptography doesn't.

C2Pay is a mobile SDK for adaptive multi-factor authentication (MFA) using device trust, biometrics, and C2PA attestation to verify high-risk payments securely. In an era where generative AI can fabricate convincing digital content, C2Pay ensures transaction authenticity through cryptographic verification.

The SDK provides risk-based authentication that only escalates verification when needed, minimizing friction for legitimate users while maintaining robust security. It integrates seamlessly into checkout flows with hardware-backed security and provides verifiable proofs for merchants.

The Problem

How do we trust digital transactions in a world of synthetic data?

The rapid rise of generative AI tools like Sora highlighted how convincingly digital content can be fabricated. This sparked a critical question: in a world where seeing is no longer believing, how can we establish trust in digital transactions?

Traditional payment authentication creates friction for users while still being vulnerable to sophisticated fraud. We needed a solution that could verify authenticity without compromising user experience‚Äîone that leverages cryptographic proof rather than blind trust.

C2PA cryptographic receipt ensuring transaction authenticity

The Solution

Adaptive, risk-based authentication with cryptographic verification

"Instead of treating every transaction the same, C2Pay intelligently assesses risk and only escalates authentication when necessary‚Äîbacked by cryptographic proof."

C2Pay collects device signals, behavioral patterns, and biometrics to calculate a real-time risk score. Only when the score exceeds a defined threshold does authentication escalate via C2PA manifests and TEE-backed device keys, creating verifiable transaction records.

The final C2Pay SDK demonstrates how cryptographic verification can be both powerful and unobtrusive. For low-risk transactions, users experience frictionless checkout. When risk increases, additional verification happens transparently, with all transactions creating verifiable audit trails.

Challenges & Solutions

Balancing security with user experience

Challenge: Our initial focus on AI-generated image detection seemed too narrow. Pivoting to financial security mid-hackathon required rapid learning and restructuring.

Solution: We leveraged our encryption and verification research, realizing the same principles could directly improve fintech security. This pivot actually strengthened our submission by addressing a more immediate market need.

Challenge: Integrating TEE and C2PA verification in a lightweight mobile library while maintaining developer-friendly APIs.

Solution: We created abstraction layers that handle complex cryptographic operations behind simple SDK methods, allowing developers to integrate robust security with minimal code.

What's Next

Evolution to production-ready SDK

C2Pay will evolve into a cross-platform SDK that developers can integrate into any checkout flow with minimal effort. Future development includes:

  • Enhanced ML Models: Real-time fraud detection with machine learning-powered risk assessment
  • Merchant Dashboard: Visualization tools for fraud patterns and transaction analytics
  • Universal Verification: Creating a standard layer where every transaction carries cryptographic proof
  • Cross-Platform Support: Expanding beyond mobile to web and IoT payment devices

Long-term, C2Pay aims to establish a new standard where every online transaction carries verifiable proof of device integrity and user authenticity‚Äîmaking "people lie, cryptography doesn't" the foundation of digital commerce.

What I Learned

Key takeaways from building C2Pay

Pivoting is powerful: Our willingness to shift from AI image detection to financial security mid-hackathon led to a stronger, more impactful solution. Sometimes the best ideas emerge from asking "what else could this solve?"

Security doesn't have to mean friction: Through risk-based authentication, we learned that the strongest security can also be the most invisible‚Äîappearing only when needed.

Developer experience matters: Creating an SDK taught me the importance of abstraction and documentation. Complex cryptographic operations need to be accessible through simple, intuitive APIs.

Standards matter: Learning about C2PA and TEE revealed how industry standards enable innovation. Building on established protocols accelerates development and ensures interoperability.
